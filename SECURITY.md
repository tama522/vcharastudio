# Security Policy

## Supported Versions

The main branch is the only supported line unless maintainers document a release branch.

## Reporting a Vulnerability

Please report vulnerabilities through GitHub Security Advisories when enabled on the repository. If advisories are not available yet, open a private contact channel with the maintainer before disclosing details publicly.

Do not include real user data, private images, credentials, or active Codex worker tokens in public issues.

## Scope

Important areas include:

- Authentication and session handling
- Per-user asset authorization
- D1 data isolation
- R2 object access
- Codex worker token creation, expiry, revocation, and job claiming
- Local development sign-in controls
